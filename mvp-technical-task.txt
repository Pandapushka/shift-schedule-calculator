Техническое задание: MVP копия сайта grafik-smen.ru на ASP.NET Core + Vue.js + PostgreSQL

1. Цель проекта (MVP)
- Создать минимально жизнеспособную копию сайта grafik-smen.ru с фокусом на калькулятор графика смен.
- Платформа фронтенда: Vue.js (SPA).
- Платформа бэкенда: ASP.NET Core Web API.
- База данных: PostgreSQL (опционально, можно начать с in-memory или SQLite для MVP).
- Архитектурный паттерн бэкенда: Clean Architecture / Onion Architecture.

2. Общая структура MVP
- Главная страница: калькулятор графика смен.
- Убрать дополнительные калькуляторы (зарплата, отпускные и т.д.) — реализовать в будущих итерациях.
- Убрать разделы статей и истории расчётов — добавить позже.

3. Функциональные требования (MVP)

3.1. Главный калькулятор графика смен
- Форма ввода:
  - дата первого рабочего дня
  - количество рабочих дней в цикле (1-7)
  - количество выходных дней в цикле (1-7)
  - длительность смены в часах (1-24)
  - количество месяцев для расчета (1-24)
  - флаг "первая смена — ночь"
  - название графика (опционально)
- Логика расчета:
  - построить календарь на заданное количество месяцев вперед, начиная с указанной даты.
  - отметить рабочие смены классом `work`.
  - свободные дни оставлять пустыми.
  - учитывать длину смены для подсчета общего количества часов.
- Результат:
  - календарные таблицы по месяцам (адаптивные)
  - статистика: количество смен и часов в каждом месяце
  - общий результат по всем месяцам

4. Архитектура бэкенда (MVP)

4.1. Паттерн: Clean Architecture / Onion Architecture
- Слои приложения:
  - Domain: бизнес-сущности (ShiftSchedule, DayStatus), интерфейсы репозиториев.
  - Application: сервисы (IShiftScheduleService), команды/запросы, DTO.
  - Infrastructure: реализация репозиториев (in-memory для MVP, позже PostgreSQL).
  - Web/API: контроллеры (ShiftScheduleController), маршрутизация.

4.2. Компоненты бэкенда
- Контроллер API: `ShiftScheduleController`
  - POST /api/shiftschedule/calculate
- Сервис: `IShiftScheduleService` и реализация
- Репозиторий: `IShiftScheduleRepository` для сохранения результатов
- DTO: ShiftScheduleRequest, ShiftScheduleResponse.

5. База данных (MVP)
- PostgreSQL обязательна.
- Таблица `shift_schedules`:
  - id (primary key)
  - title
  - start_date
  - work_days
  - off_days
  - hours_per_shift
  - months
  - night_first
  - calendar_json (JSON с результатом)
  - created_at
- Настроить EF Core Migrations для PostgreSQL.
- Сохранять результаты расчётов в базу.

6. Фронтенд на Vue.js (MVP)

6.1. Структура проекта
- `src/components`:
  - `ShiftScheduleForm.vue`
  - `ScheduleCalendar.vue`
- `src/pages`:
  - `HomePage.vue`
- `src/router/index.js`
- `src/store` (Pinia): состояние результатов.

6.2. UI/UX
- Простая форма на главной странице.
- Результат отображать в виде карточек-таблиц, похожих на оригинал.
- Адаптивность: календарь в один столбец на мобильных.
- Без меню и навигации — только форма и результат.

6.3. API взаимодействие
- POST /api/shiftschedule/calculate с данными формы.
- Отображение результата после ответа.

7. Детали реализации главного калькулятора

7.1. Алгоритм
- Вычислить последовательность смен по циклу `work_days` / `off_days`.
- Начало берётся с `start_date`.
- Если флаг `night_first` активен, первая работа считается как ночная (визуально).
- Для каждого дня статус: `work`, `off`, `empty`.
- Подсчитать смены и часы.

7.2. Ответ API
- JSON:
  {
    "title": "...",
    "months": [
      {
        "year": 2026,
        "month": 4,
        "days": [
          { "day": 1, "status": "work" },
          { "day": 2, "status": "work" },
          { "day": 3, "status": "off" },
          ...
        ],
        "workCount": 16,
        "hoursCount": 176
      }
    ],
    "totalWorkCount": 62,
    "totalHours": 682
  }

- Vue.js 3.
- Для MVP: без базы данных, in-memory.
- CORS для фронтенда.
