Техническое задание: копия сайта grafik-smen.ru на ASP.NET Core + Vue.js + PostgreSQL

1. Цель проекта
- Создать рабочую копию сайта grafik-smen.ru.
- Платформа фронтенда: Vue.js (SPA или упрощённый мультистраничный интерфейс).
- Платформа бэкенда: ASP.NET Core Web API.
- База данных: PostgreSQL.
- Архитектурный паттерн бэкенда: Clean Architecture / Onion Architecture.

2. Общая структура копии
- Главная страница: калькулятор графика смен.
- Дополнительные страницы:
  - калькулятор зарплаты (`/zp`)
  - калькулятор отпускных (`/otpusk`)
  - калькулятор потребительского кредита (`/potreb_calc`)
  - калькулятор разницы дат (`/calc_dat`)
  - калькулятор процентов (`/calc_proc`)
  - калькулятор НДС (`/calc_nds`)
  - статьи (`/art`)
  - история расчётов (`/spis`)

3. Функциональные требования

3.1. Главный калькулятор графика смен
- Форма ввода:
  - дата первого рабочего дня
  - количество рабочих дней в цикле
  - количество выходных дней в цикле
  - длительность смены в часах
  - количество месяцев для расчета
  - флаг "первая смена — ночь"
  - название графика
- Логика расчета:
  - построить календарь на заданное количество месяцев вперед, начиная с указанной даты.
  - отметить рабочие смены классом `work`.
  - свободные дни оставлять пустыми.
  - учитывать длину смены для подсчета общего количества часов.
- Результат:
  - календарные таблицы по месяцам
  - статистика: количество смен и часов в каждом месяце
  - общий результат по всем месяцам

3.2. Другие калькуляторы
- Для первой версии можно реализовать статические заглушки или простые интерфейсы.
- В идеале на бэкенде реализовать отдельные API-эндпоинты для каждого калькулятора.
- Данные форм могут быть сохранены в истории.

3.3. История расчетов
- Сохранять запросы калькуляторов и результаты в базе PostgreSQL.
- Отображать в `/spis` список выполненных расчётов с датой, типом и кратким описанием.

3.4. Статьи
- Страница `/art` должна отображать карточки статей.
- Для первой версии можно использовать статические записи в базе или JSON.
- Каждая статья должна вести на собственную страницу (по пути `/art/{slug}`).

4. Архитектура бэкенда

4.1. Паттерн: Clean Architecture / Onion Architecture
- Слои приложения:
  - Domain: бизнес-сущности, интерфейсы репозиториев, доменные правила.
  - Application: сервисы, команды, запросы, DTO, обработчики, бизнес-логика.
  - Infrastructure: реализация доступа к PostgreSQL, механизмы логирования, работа с внешними сервисами.
  - Web/API: контроллеры, маршрутизация, конфигурация ASP.NET Core.
- Важно: зависимости идут внутрь, слой `Web/API` зависит от `Application`, `Application` зависит от `Domain`, `Infrastructure` зависит от `Application/Domain`.

4.2. Почему этот паттерн подходит
- Проект должен быть расширяемым: добавление новых калькуляторов, хранения истории, статьи.
- Чёткое разделение ответственности между UI, логикой и данными.
- Лёгкое тестирование бизнес-логики без зависимости от инфраструктуры.
- PostgreSQL интерфейс реализуется через репозитории и миграции.

4.3. Компоненты бэкенда

7. Детали реализации главного калькулятора

7.1. Алгоритм
- Вычислить последовательность смен по циклу `work_days` / `off_days`.
- Начало берётся с `start_date`.
- Если флаг `night_first` активен, первая работа считается как ночная смена (визуально / меткой).
- Для каждого дня добавить в календарь статус:
  - `work` — рабочая смена
  - `` — выходной день
  - `empty` — ячейка для дней до начала месяца или после конца месяца.
- Подсчитать количество рабочих смен и часов в каждом месяце.

7.2. Ответ API
- Формат JSON:
  {
    "title": "...",
    "months": [
      {
        "year": 2026,
        "month": 4,
        "days": [
          { "day": 1, "status": "work" },
          { "day": 2, "status": "work" },
          { "day": 3, "status": "off" },
          ...
        ],
        "workCount": 16,
        "hoursCount": 176
      }
    ],
    "totalWorkCount": 62,
    "totalHours": 682
  }

8. Требования к инфраструктуре
- ASP.NET Core 8+.
- EF Core с поддержкой PostgreSQL.
- Vue.js 3.
- Релизный билд фронтенда собирается и деплоится как статические файлы в ASP.NET Core или разворачивается отдельно.
- Автозагрузка конфигурации через `appsettings.json`.
- Безопасность: CORS для фронтенд-домена, защита от XSS и CSRF при необходимости.

9. Дополнительные требования
- Минимизировать рекламные блоки в копии (не нужно копировать исходную рекламу).
- Сделать код чистым, документированным и расширяемым.
- Опционально: реализовать простую CMS-структуру для статей.

- Создать Vue.js приложение и базовый роутинг.
- Реализовать главный калькулятор и вывод календаря.

- Написать README с инструкциями по запуску.

11. Рекомендации по архитектуре
- Clean Architecture / Onion Architecture:
  - `Domain` — бизнес-объекты
  - `Application` — команды/запросы, интерфейсы сервисов
  - `Infrastructure` — EF Core, PostgreSQL, реализация репозиториев
  - `WebApi` — контроллеры, middleware
- Это оптимальный выбор: структура подходит для роста, легко покрывается тестами, не связана с конкретной БД.
